1. APPROVED POSITION
==================================================

Debt is approved as:

Debt is not approved to become:
- a second Overview
- a second Reporting section
- a second Atlas surface
- a generic finance dashboard
- a local recommendation or scoring engine

==================================================
2. APPROVED LAUNCH DIRECTION
==================================================

The broader launch direction is approved based on the bounded launch contract you provided.

That approval is based on the fact that you demonstrated:
- bounded scope
- clean Bills separation
- Atlas compatibility
- real end-to-end user flows
- trust-aware failure handling
- UI containment
- integration simplicity

==================================================
3. NON-NEGOTIABLE IMPLEMENTATION CONDITIONS
==================================================

Approval is granted with the following implementation conditions remaining in force:

1. Bills separation must remain clean
- Debt owns debt truth and debt math
- Bills owns operational payment rows and payment capture
- Debt must not turn Bills into a debt-management clone

2. Atlas remains the recommendation layer
- Debt must not introduce debt-local recommendation behavior
- Debt must not introduce debt-local urgency or scoring logic
- Atlas continues to consume shared truth only

3. UI containment must be protected
- Debt summary must not become a second Overview
- Debt detail views must not become Reporting clones
- Debt must remain account-centric, not dashboard-centric

4. Trust handling must remain explicit
- no fake precision
- no silent assumption drift
- no black-box math
- incomplete inputs must degrade gracefully

5. Scope discipline must be preserved
- anything listed as out / deferred remains out / deferred unless separately approved
