==================================================
2. OVERVIEW INTENDED USE
==================================================

Overview intends to surface a small set of high-level Debt facts in a compact dashboard treatment.

Overview will use Debt outputs for:
- quick-glance visibility
- high-level summary cards
- compact debt snapshot information
- routing users into Debt for details

Overview will not:
- calculate debt math
- calculate delinquency
- calculate cure status
- calculate cash-window totals
- infer urgency
- rank debt accounts
- reinterpret Debt states
- duplicate Debt detail views

Core rule:
Overview displays Debt facts. Debt defines Debt. Atlas interprets Debt.

==================================================
3. PROPOSED OVERVIEW METRICS
==================================================

Overview is currently considering the following Debt V2.2 metrics for display:

1. Total Debt
- Desired meaning: total outstanding debt balance across active debt accounts
- Display type: currency
- Likely placement: Debt Snapshot / high-level summary surface

2. Active Debt Account Count
- Desired meaning: count of active/open debt accounts
- Display type: integer
- Likely placement: supporting label or secondary metric

3. Accounts Behind
- Desired meaning: count of debt accounts that are late, delinquent, or otherwise behind according to Debt-owned standing logic
- Display type: integer
- Likely placement: Debt Snapshot / attention signal

4. Next Debt Payment
- Desired meaning: next required debt payment surfaced by Debt
- Display type: amount + due date + optional account reference
- Likely placement: Debt Snapshot

5. Debt Cash Needed Soon
- Desired meaning: Debt-owned required cash amount due within a supported future window
- Display type: currency + window label
- Likely placement: Debt Snapshot / short-term cash context

6. Accounts Requiring Review
- Desired meaning: count of accounts Debt has flagged for user review
- Display type: integer
- Likely placement: optional Debt Snapshot item or warning label

7. Data Quality / Source Flags
- Desired meaning: counts or labels for stale, limited-confidence, or source-conflicted Debt records
- Display type: compact badge/count
- Likely placement: supporting context only

5. SPECIFIC QUESTIONS
==================================================

A) Total Debt
1. What is the exact field name?
2. Does it include only active/open accounts?
3. Does it include closed-with-balance accounts?
4. Are collections/settlement accounts included?
5. Are disputed or limited-confidence balances included?

B) Accounts Behind
1. What is the exact field name?
2. Which Debt standing states count as behind?
3. Does this include delinquent, missed, collections, settlement, or default states?
4. Is the count fully computed by Debt?
5. Should Overview show the count only, or also expose the included state labels?

C) Next Debt Payment
1. What is the exact field name or object shape?
2. Does Debt provide amount, date, account ID, account label, and schedule ID?
3. What happens if multiple payments share the same next date?
4. What happens if there are no scheduled debt payments?
5. Should Overview route to the Debt account, Debt schedule, or related Bills row?

D) Debt Cash Needed Soon
1. What is the exact field name?
2. What windows are supported? 14 / 30 / 60 days?
3. Which window does Debt recommend Overview use by default?
4. What qualifies as required debt cash?
5. Are partial payments, cure amounts, failed/reversed payments, or arrangements included?
6. Is this fully calculated by Debt?
7. What label should Overview use so it does not imply local urgency logic?

E) Accounts Requiring Review
1. What is the exact field name?
2. What conditions trigger review-required status?
3. Is this deterministic or judgment-based?
4. What supporting reason labels are available?
5. Where should the user be routed from this signal?

F) Data Quality / Source Flags
1. What fields expose limited-confidence, stale, or source-conflicted Debt data?
2. Are these counts, booleans, enums, or per-account flags?
3. What wording should Overview preserve?
4. Should these be shown only when non-zero?
